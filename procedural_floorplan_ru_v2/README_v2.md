# procedural_floorplan_ru_v2

Новый аддон живет параллельно `procedural_floorplan_ru` и не заменяет его.

## Архитектура

- `building_manager.py`: оркестратор single-story генерации. Собирает `GenerationContext`, готовит footprint и запускает builders.
- `building_stories_manager.py`: оркестратор multi-story генерации. Строит базовый footprint, планирует outline каждого этажа и затем запускает builders по story context.
- `builders/base_builder.py`: единый контракт для строителей.
- `builders/floor_builder.py`: текущий builder, который создает только floor tiles `1x1 м`.
- `shape_generator.py`: дискретный генератор footprint по формам. Возвращает набор тайлов, а не Blender mesh.
- `planning/vertical_profile_planner.py`: отдельный planner вертикального профиля. Работает на уровне 2D footprint и выдает `story_plan.footprint` для каждого этажа.
- `atlas.py`: совместимая основа работы с `manifest.json`, atlas image, редактором тайлов и применением материалов/UV.
- `props.py`, `ui.py`, `operators.py`: UI-слой и Blender registration.

## Как добавлять новые builders

1. Добавить новый класс в `builders/`, унаследованный от `BaseBuilder`.
2. Реализовать `builder_id`, `enabled(context)` при необходимости и `build(context)`.
3. Зарегистрировать builder в `BuildingManager.register_default_builders()`.

## Как запускается генерация

1. Пользователь задает свойства в UI.
2. Оператор `floorplan_ru_v2.generate` собирает `GenerationSettings`.
3. `BuildingStoriesManager` строит базовый footprint по `shape_mode`.
4. `VerticalProfilePlanner` создает список `story_plan.footprint` по `vertical_profile_mode`.
5. Builders строят каждый этаж уже по его собственному итоговому footprint.
6. Если атлас включен, `atlas.apply_atlas_to_collection()` назначает material и UV.

## Интерфейс

### Кнопки вверху

- `Сгенерировать дом`: запускает генерацию нового результата по текущим настройкам. На этом этапе создает только пол.
- `Сбросить настройки`: возвращает свойства `procedural_floorplan_ru_v2` к значениям по умолчанию.

### Блок `Общее`

- `Удалять старое`: перед новой генерацией очищает предыдущие объекты из целевой коллекции.
- `Имя коллекции`: имя Blender-коллекции, в которую складываются все объекты нового генератора v2.
- `Seed`: зерно генерации. При одинаковых параметрах и одинаковом `Seed` footprint должен получаться одинаковым.
- `Количество комнат`: пока не создает реальные комнаты. Сейчас влияет на общий размер и сложность footprint.
- `Масштаб дома`: дополнительный множитель размера footprint.
- `Размер подписей`: зарезервировано под будущие подписи и вспомогательные обозначения. На этапе генерации пола почти не влияет.

### Блок `Форма дома`

- `Прямоугольник`: простой прямоугольный footprint.
- `L`: Г-образный footprint с одним вырезом.
- `U`: U-образный footprint с внутренним вырезом.
- `H`: footprint с двумя боковыми крыльями и центральной перемычкой.
- `T`: footprint в форме буквы T.
- `Курдонер`: форма с центральным парадным вырезом и двумя передними крыльями.
- `Смещенный`: составной footprint, в котором верхняя или боковая часть смещена относительно нижней, но форма остается связной.

### Блок `Этажность`

- `Количество этажей`: сколько этажей будет сгенерировано.
- `Режим планировки этажей`: одинаковый room layout на всех этажах или независимая генерация по story seed.
- `Вертикальный профиль`: как меняется footprint между этажами.
- `STRICT`: старое поведение. Каждый этаж точно повторяет базовый outline.
- `SETBACK`: верхние этажи постепенно отступают от части внешних граней и формируют ступенчатый силуэт.
- `OFFSET_STACK`: этажи сохраняют форму, но смещаются относительно нижних в пределах безопасного overlap.
- `PINWHEEL`: этажи по кругу меняют доминирующее направление вокруг защищенного центрального ядра.
- `MIXED`: детерминированно комбинирует strict, setback, offset и pinwheel-логику.
- `Сила профиля`: ограничивает глубину setback и величину floor-to-floor смещения.

## Валидность вертикального профиля

Перед тем как новый outline этажа попадет в builders, `VerticalProfilePlanner` проверяет:

- связность footprint;
- минимальные габариты и минимальную площадь;
- сохранение центрального protected core под лестницу/холл;
- минимальный overlap с предыдущим этажом для режимов смещения;
- безопасный fallback: если модификация невалидна, этаж повторяет предыдущий валидный footprint, а генерация не падает.

### Блок `Атлас`

- `Включить атлас`: включает использование `manifest.json` и atlas image для назначения материалов и UV.
- `Путь к manifest.json`: путь к atlas manifest. Поддерживается Blender-формат путей, например `//house_atlas.json`.
- `Путь к изображению атласа`: путь к atlas image. Если поле пустое, путь берется из `meta.source_image` внутри manifest.
- `Включать внутренние стены`: совместимое свойство из старого аддона. На текущем этапе генерации пола фактически не используется.
- `Случайный выбор тайла`: если у объекта нет принудительного `atlas_tile_id`, тайл будет выбираться из категории автоматически.

### Кнопки блока `Атлас`

- `Загрузить manifest.json`: читает manifest с диска и заполняет editor state в UI.
- `Сохранить manifest.json`: сохраняет изменения редактора тайла обратно в manifest на диске.
- `Применить атлас`: применяет материал и UV к уже созданному полу без полной регенерации.

### Подблок `Настройки текстур из manifest.json`

- `Окна`: совместимые placement-настройки из старого аддона.
- `Смещение X`, `Смещение Y`: сдвиг текстуры окна внутри логики placement.
- `Ширина`, `Высота`: масштабы placement для окна.
- `Двери`: совместимые placement-настройки для дверных тайлов.
- `Смещение X`, `Смещение Y`: сдвиг текстуры двери внутри логики placement.
- `Ширина`, `Высота`: масштабы placement для двери.

На текущем этапе эти настройки сохраняются и поддерживаются ради совместимости формата `manifest.json`, но сам `FloorBuilder` их не использует, потому что строится только пол.

### Подблок `Редактор выбранного тайла`

- `Категория`: категория тайлов внутри manifest, например `floors`, `walls`, `roofs`.
- `Тайл`: индекс и id текущего выбранного entry внутри выбранной категории.
- `ID тайла`: строковый идентификатор tile entry.
- `X`, `Y`: позиция тайла в atlas image в пикселях.
- `W`, `H`: размер региона тайла в atlas image в пикселях.
- `Tile Width (m)`: ожидаемая ширина покрытия этого тайла в метрах.
- `Tile Height (m)`: ожидаемая высота или глубина покрытия этого тайла в метрах.

Для текущего этапа практически важнее всего категория `floors`, потому что именно она назначается всем `FloorTile_*`.

## Ограничение текущего этапа

Текущая версия уже использует per-story footprint во всем основном pipeline builders, но сложные вертикальные режимы пока ограничены стратегиями `STRICT`, `SETBACK`, `OFFSET_STACK`, `PINWHEEL` и `MIXED`. Режимы с внутренним двором и более агрессивной сменой типологии по этажам можно добавлять дальше как отдельные стратегии planner-слоя.
